# Business Documentation

## Product Overview

Event.me is a simplified scheduling platform that enables professionals to share their availability and allow others to book meetings with them - similar to Calendly, but focused on simplicity and ease of use.

## Key Value Propositions

1. **Simple Setup**: Get started in minutes by defining your available hours
2. **Professional Booking Pages**: Share clean, branded booking links
3. **No Back-and-Forth**: Let people book directly into your calendar
4. **Multiple Meeting Types**: Offer different types of meetings with various durations

## Target Users

- **Freelancers**: Consultants, coaches, and service providers
- **Sales Teams**: Sales representatives scheduling demos and calls
- **Recruiters**: HR professionals coordinating interviews
- **Educators**: Teachers and tutors offering office hours
- **Anyone**: Professionals who schedule meetings regularly

## Core Features

### 1. User Registration & Authentication
**User Benefit**: Secure account creation with email and password

**How it Works**:
- Sign up with name, email, username, and password
- Username becomes part of your booking URL
- Secure password storage with industry-standard encryption

### 2. Event Type Management
**User Benefit**: Create different types of meetings you offer

**Use Cases**:
- 15-minute quick call
- 30-minute consultation
- 60-minute strategy session
- 45-minute job interview

**Key Features**:
- Custom title and description
- Set meeting duration
- Unique URL slug for sharing
- Toggle active/inactive status

### 3. Availability Settings
**User Benefit**: Define when you're available for bookings

**How it Works**:
- Set available hours for each day of the week
- Example: Monday 9:00 AM - 5:00 PM
- Add multiple time blocks per day
- Manage availability through intuitive dashboard

**Flexibility**:
- Different hours for different days
- Easy to add/remove availability slots
- Changes reflect immediately on booking pages

### 4. Public Booking Pages
**User Benefit**: Share professional booking links with anyone

**URL Structure**: `event.me/{your-username}/{meeting-type}`

**Booking Experience**:
1. Guest visits your booking link
2. Sees meeting description and duration
3. Selects available date from calendar
4. Chooses from available time slots
5. Enters their information (name, email, notes)
6. Confirms booking

**Features**:
- Clean, professional design
- Mobile-friendly interface
- Real-time availability display
- Instant booking confirmation

### 5. Booking Management
**User Benefit**: Track and manage your scheduled meetings

**Capabilities**:
- View all upcoming bookings
- See guest information
- Check booking details
- Prevent double-bookings automatically

## User Workflows

### New User Onboarding
1. Visit Event.me homepage
2. Click "Get Started"
3. Create account with email/username
4. Set up availability
5. Create first event type
6. Share booking link

### Scheduling a Meeting (Guest Perspective)
1. Receive booking link
2. View meeting details
3. Select date
4. Choose time slot
5. Provide contact information
6. Confirm booking
7. Receive confirmation

### Managing Meetings (User Perspective)
1. Log in to dashboard
2. View upcoming bookings
3. Check availability status
4. Update event types as needed
5. Adjust availability when schedule changes

## Dashboard Overview

The dashboard is the control center with three main sections:

1. **Overview**: Summary of bookings and event types
2. **Event Types**: Manage your meeting types
3. **Availability**: Set your available hours

### Common Questions

**Q: How do I share my booking link?**
A: Copy the URL from your event type and share via email, website, or social media.

**Q: Can I have different availability for different meeting types?**
A: Currently, availability applies to all event types. Different schedules coming in Phase 2.

**Q: What happens if someone books outside my availability?**
A: The system only shows available time slots based on your settings.

**Q: Can I cancel or reschedule bookings?**
A: This feature is planned for Phase 2.
